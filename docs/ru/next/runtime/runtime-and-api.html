Runtime и API

Все чейны Substrate имеют специальную среду выполнения, т. н. runtime. Runtime — это двоичный файл, который содержит байткод WebAssembly и включает набор заранее известных точек входа. Ключевыми являются две точки входа, заявленные как часть спецификации Substrate: execute_block и version. Точки входа сгруппированы в наборы фич и представляются т. н. API типажами (в Rust, по крайней мере). В частности, две названные ранее обязательные точки входа сгруппированы в месте в API типаже Core.

API помимо Core могут быть предоставлены по желанию для расширения функциональности блокчейн клиента. Существует набор стандартных API который предоставляет функциональность распространенных (хотя и опциональных) клиентских компонентов, включая модули обработки метаданных, авторинга блоков и очередей транзакций.

Объявление API

Для рантаймов, написанных с использованием языка Rust, существует специальный макрос impl_apis, который автоматически предоставляет необходимые маршаллинг и диспетчеризацию и, тем самым, упрощает регистрацию новых точек входа. В то же время рантаймы, написанные на других языках, будут вынуждены реализовывать такую логику самостоятельно. Сюда относится кодирование/декодирование типов данных, управление передачей параметров и возвращаемых значений посредством предварительно выделенной памяти, и выставления соответствующих точек входа в качестве публичного API. (Существует специальный API, основанный на Substrate Simple Codec (codec), который используется для кодирования и декодирования аргументов функций, а также для указания того как и когда они должны передаваться.)
