The subkey Tool

Subkey — это утилита для создания ключей, которая разрабатывается вместе с Substrate. Его основные функции - генерация sr25519 и ed25519 пар ключей, кодирование SS58 адресов и восстановление ключей из мнемонических фраз и необработанных сидов. Также она может создавать и проверять подписи, в том числе для закодированных транзакций.

Создание бинарного файла Subkey

Однострочная установка

Бинарный файл Subkey, subkey, также установливается вместе с Установкой Substrate. Если Вы хотите поиграть только с Subkey (без Substrate), Вам нужно будет иметь зависимости Substrate. Используйте следующие две команды для установки зависимостей и Subkey, соответственно:

$ curl https://getsubstrate.io -sSf | bash -s -- --fast
$ cargo install --force --git https://github.com/paritytech/substrate subkey

Компилирование с Cargo

Если у Вас уже есть репозиторий Substrate, Вы можете собрать Subkey с помощью:

$ cargo build -p subkey

Это установит subkey в ./target/debug/subkey.

Создание ключей

Сгенерировать sr25519 ключ, запустив:

$ subkey generate

Secret phrase `favorite liar zebra assume hurt cage any damp inherit rescue delay panic` is account:
  Secret seed: 0x235c69907d33b85f27bd78e73ff5d0c67bd4894515cc30c77f4391859bc1a3f2
  Public key (hex): 0x6ce96ae5c300096b09dbd4567b0574f6a1281ae0e5cfe4f6b0233d1821f6206b
  Address (SS58): 5EXWNJuoProc7apm1JS8m9RTqV3vVwR9dCg6sQVpKnoHtJ68

Для еще большей безопасности используйте --words 24 (поддерживается 12, 15, 18, 21 и 24):

$ subkey generate --words 24
Secret phrase `engine ghost pave tip coil undo when next finish between ignore mystery spread model mercy body sphere sense verify crumble ethics garment soon gold` is account:
  Secret seed: 0xddd41c017dd60770a48f7aa50aee940b591726117769bc0740b19c5acfbf24d8
  Public key (hex): 0x8ae95a30cb9daa11db449eeb3243ec2e8937df3620c607d66a7f330a9d48cf01
  Address (SS58): 5FCqnWDeL8KFEhy9ccdeRw64kJbfPLfKgA45kMeo6THj1tkz

Subkey кодирует адрес в зависимости от сети. Вы можете использовать -n или --network флаг для этого изменения. Смотрите subkey help для поддерживаемых сетей.

Чтобы сгенерировать ключ ed25519, передайте -e или --ed25519 флаг:

$ subkey -e generate
Secret phrase `expire stage crawl shell boss any story swamp skull yellow bamboo copy` is account:
  Secret seed: 0xb034ad2704defa9dc0bea4ac8019fb0f805018f1be587cf04a03f9a033e3656b
  Public key (hex): 0x3ff0766f9ebbbceee6c2f40d9323164d07e70c70994c9d00a9512be6680c2394
  Address (SS58): 5DWYJiPBSFBUFah2W49oPPSsrCRvrY4N4VmLcSH9XfjKfZvh

Вывод дает нам следующую информацию о нашем ключе:

  • Secret phrase (Секретная фраза, а также Мнемоническия фраза) - серия английских слов, которые кодируют сиды в формате более понятном человеку. Мнемонические фразы впервые были введены в биткоин (см. BIP39) и сделали намного проще запись своего ключа вручную.
  • Secret Seed ("Секретный сид", а также "Приватный ключ" или "Сырой Seed") - Минимальная необходимая информация для восстановления ключевой пары. Вся другая информация рассчитывается из сида.
  • Public Key (hex) - Публичная половина криптографической пары ключей в шестнадцатеричном.
  • Address (SS58) (также "Публичный адрес") - SS58-кодированный адрес на основе открытого ключа.

Вы также можете создать vanity адрес, хотя Вы не получите мнемоническую фразу:

$ subkey vanity joe
Generating key containing pattern 'joe'
best: 189 == top: 189
Secret Key URI `0xc645213712dc218e851ac0a47e987ee3f8e1cbbbad85522dbbf86e51963de434` is account:
  Public key (hex): 0xfaff3b483564c4d2dab7229e4369841d3650a4ca9c36094e9faece3839ac6e06
  Address (SS58): 5HjoeXxUokiJwQPRucDXCgGzKwwf74q9srJcmsaFssUW2UHn

Защищенные паролем ключи

Чтобы сгенерировать защищенный паролем ключ, используйте -p <password> или --password <password> флаг:

$ subkey --password "correct horse battery staple" generate
Secret phrase `razor blouse enroll maximum lobster bacon raccoon ocean law question worry length` is account:
  Secret seed: 0x806e324821f6b09fa0329b7da35b8056b452fdcaadeac463ff2b8db1fc020c3e
  Public key (hex): 0xe2b410bdc959fcc7f756e6eefdb051000c80be339acd3a8dc6d941211160b266
  Address (SS58): 5HBxAo81NWBbg6DQNbTdFg7GWQDS1QmkMwNLQso1e11DfQ2Y

Обратите внимание, что "Secret seed" является не защищенным паролем и может по-прежнему восстановить учетную запись. Однако "Secret phrase" недостаточно для восстановления учетной записи без пароля.

Проверка ключей

Команда inspect пересчитывает открытый ключ ключевой пары и публичный адрес с учетом ее seed. Это показывает, что для резервной копии достаточно только сида.

$ subkey inspect 0x235c69907d33b85f27bd78e73ff5d0c67bd4894515cc30c77f4391859bc1a3f2
Secret Key URI `0x235c69907d33b85f27bd78e73ff5d0c67bd4894515cc30c77f4391859bc1a3f2` is account:
  Public key (hex): 0x6ce96ae5c300096b09dbd4567b0574f6a1281ae0e5cfe4f6b0233d1821f6206b
  Address (SS58): 5EXWNJuoProc7apm1JS8m9RTqV3vVwR9dCg6sQVpKnoHtJ68

Вы также можете проверить ключ своей мнемонической фразой.

$ subkey inspect "favorite liar zebra assume hurt cage any damp inherit rescue delay panic"
Secret phrase `favorite liar zebra assume hurt cage any damp inherit rescue delay panic` is account:
  Secret seed: 0x235c69907d33b85f27bd78e73ff5d0c67bd4894515cc30c77f4391859bc1a3f2
  Public key (hex): 0x6ce96ae5c300096b09dbd4567b0574f6a1281ae0e5cfe4f6b0233d1821f6206b
  Address (SS58): 5EXWNJuoProc7apm1JS8m9RTqV3vVwR9dCg6sQVpKnoHtJ68

Вы можете проверить защищенные паролем ключи либо, подставив --password флаг или используя /// в конце мнемоники:

$ subkey inspect "razor blouse enroll maximum lobster bacon raccoon ocean law question worry length///correct horse battery staple"
Secret Key URI `razor blouse enroll maximum lobster bacon raccoon ocean law question worry length///correct horse battery staple` is account:
  Public key (hex): 0xe2b410bdc959fcc7f756e6eefdb051000c80be339acd3a8dc6d941211160b266
  Address (SS58): 5HBxAo81NWBbg6DQNbTdFg7GWQDS1QmkMwNLQso1e11DfQ2Y

Давайте предположим, что Вы хотите использовать тот же приватный ключ на Polkadot. Используйте -n , чтобы получить Ваш адрес в формате Polkadot. Обратите внимание, что публичный ключ одинаковый, но адрес имеет другой формат.

$ subkey --network polkadot inspect "favorite liar zebra assume hurt cage any damp inherit rescue delay panic"
Secret phrase `favorite liar zebra assume hurt cage any damp inherit rescue delay panic` is account:
  Secret seed: 0x235c69907d33b85f27bd78e73ff5d0c67bd4894515cc30c77f4391859bc1a3f2
  Public key (hex): 0x6ce96ae5c300096b09dbd4567b0574f6a1281ae0e5cfe4f6b0233d1821f6206b
  Address (SS58): 13ToWeAsFe55Z7qGxwV8uJFch73aCEyHhhQb2hVAsspp4Xuo

HD деривация

Subkey поддерживает жесткую и мягкую иерархическую детерминированную (HD) деривацию ключей, совместимую с BIP32. HD-ключи позволяют Вам иметь мастер-ключ и определить дерево с ключевой парой на каждом листке. Дерево имеет структуру аналогичную файловой системе и может иметь любую глубину, которую Вы можете получить.

Поддерживается жесткая и мягкая деривация ключей:

  • родительский закрытый ключ + путь --> дочерний закрытый ключ
  • родительский закрытый ключ + путь --> дочерний открытый ключ

Кроме того, поддерживаются производные ключи:

  • родительский открытый ключ + путь --> дочерний открытый ключ

Получение жесткого ключа

Вы можете получить дочерний жесткий ключ, используя // после мнемонической фразы:

$ subkey inspect "favorite liar zebra assume hurt cage any damp inherit rescue delay panic//joe//polkadot//0"
Secret Key URI `favorite liar zebra assume hurt cage any damp inherit rescue delay panic//joe//polkadot//0` is account:
  Public key (hex): 0xb660386488b17fb0989204820ab43c2eae3053f2882dede1688ce9ee48bd0e0a
  Address (SS58): 5GBq9ybWHynecioAmQtuBXjw7vKwzhm72mURRmK2sDpsBRdk

Получение мягкого ключа

Точно так же, Вы можете получить дочерний мягкий ключ, используя один / после мнемонической фразы:

$ subkey inspect "favorite liar zebra assume hurt cage any damp inherit rescue delay panic/joe/polkadot/0"
Secret Key URI `favorite liar zebra assume hurt cage any damp inherit rescue delay panic/joe/polkadot/0` is account:
  Public key (hex): 0x34ba6dcb945ff69d1457d1488d6b506738f8ce0a5a3e838ed91a5fa813624272
  Address (SS58): 5DFqjBd9BEsLaRdSqfWiDnPvomu5VaDmW4u5YWb1d176hTfC

Повторите адрес из той же сид фразы, 5EXWNJuoProc7apm1JS8m9RTqV3vVwR9dCg6sQVpKnoHtJ68. Мы можем использовать это для получения того же дочернего адреса.

$ subkey inspect 5EXWNJuoProc7apm1JS8m9RTqV3vVwR9dCg6sQVpKnoHtJ68/joe/polkadot/0
Public Key URI `5EXWNJuoProc7apm1JS8m9RTqV3vVwR9dCg6sQVpKnoHtJ68/joe/polkadot/0` is account:
  Network ID/version: substrate
  Public key (hex): 0x34ba6dcb945ff69d1457d1488d6b506738f8ce0a5a3e838ed91a5fa813624272
  Address (SS58): 5DFqjBd9BEsLaRdSqfWiDnPvomu5VaDmW4u5YWb1d176hTfC

Обратите внимание, что два адреса здесь совпадают. Это не относится к жесткому ключу деривации.

Собираем все это вместе

Вы можете смешивать и сопоставлять жесткие и мягкие ключевые пути (хотя это не имеет большого смысла иметь жесткие пути как детей мягких путей). Например:

$ subkey inspect "favorite liar zebra assume hurt cage any damp inherit rescue delay panic//joe//polkadot/0"
Secret Key URI `favorite liar zebra assume hurt cage any damp inherit rescue delay panic//joe//polkadot/0` is account:
  Public key (hex): 0xd09ab65c743e91b30d024469e8a8b823a3aa7e8f5b4791187adf531ac2af140f
  Address (SS58): 5GnDmyt7qnEN4esrLNRtjM7xHyRe4jQsbbAHpaNSsPNws7EB

Первые два уровня (//joe//polkadot) являются жесткими, в то время как лист (/0) является мягким.

Чтобы использовать ключ производный с защищенным паролем ключом, добавьте пароль в конец:

$ subkey inspect "razor blouse enroll maximum lobster bacon raccoon ocean law question worry length/joe/polkadot/0///correct horse battery staple"
Secret Key URI `razor blouse enroll maximum lobster bacon raccoon ocean law question worry length/joe/polkadot/0///correct horse battery staple` is account:
  Public key (hex): 0x0816fe0689322e26cd2aa9c0dccb6c44851345e96f969ae85c8f1aec9fb4703d
  Address (SS58): 5CFK52zU59zUhC3s6mRobEJ3zm7JeXQZaS6ybvcuCDDhWwGG

$ subkey inspect 5HBxAo81NWBbg6DQNbTdFg7GWQDS1QmkMwNLQso1e11DfQ2Y/joe/polkadot/0
Public Key URI `5HBxAo81NWBbg6DQNbTdFg7GWQDS1QmkMwNLQso1e11DfQ2Y/joe/polkadot/0` is account:
  Network ID/version: substrate
  Public key (hex): 0x0816fe0689322e26cd2aa9c0dccb6c44851345e96f969ae85c8f1aec9fb4703d
  Address (SS58): 5CFK52zU59zUhC3s6mRobEJ3zm7JeXQZaS6ybvcuCDDhWwGG

Обратите внимание, что "адрес плюс путь вывода" создает тот же адрес, что и "мнемоническая фраза плюс путь вывода плюс пароль". Таким образом, вы можете раскрыть свой родительский адрес и пути вывода, не раскрывая свою мнемоническую фразу или пароль, сохраняя при этом контроль над всеми производными адресами.

Известные Ключи

Если вы работали с Substrate ранее, Вы, вероятно, столкнулись с повсеместными учетными записями для Alice, Bob и их друзей. Эти ключи не являются частными, но полезны для игры с Substrate без постоянного генерирования новых ключей. Вы можете проверить эти "известные" ключи с помощью subkey.

$ subkey inspect //Alice
Secret Key URI `//Alice` is account:
  Public key (hex): 0xd43593c715fdd31c61141abd04a99fd6822c8558854ccde39a5684e7a56da27d
  Address (SS58): 5GrwvaEF5zXb26Fz9rcQpDWS57CtERHpNehXCPcNoHGKutQY

Нет ничего особенного о имени Alice. Вы можете сделать то же самое трюк с Вашим именем. Но помните, что такие ключи не безопасны и полезны только для экспериментов.

$ subkey inspect //Joshy
Secret Key URI `//Joshy` is account:
  Public key (hex): 0x006f5081d495811a16724b317fcd70b8e42c9317da2ba1f5c36756a41fadec67
  Address (SS58): 5C5GvaAWfruRLnCJYgJrut1JovGeMMXV2VzXeiua7f691J9R

Подписание и проверка сообщений

Подписание

Вы можете подписать сообщение, передав сообщение Subkey на STDIN. Вы можете подписать либо с вашей seed, либо с мнемонической фразой.

$ echo "message" | subkey sign 0x235c69907d33b85f27bd78e73ff5d0c67bd4894515cc30c77f4391859bc1a3f2
50192c7f1a49dd169fa69f7fc703b3fe2e8c0b71871117e59b59b59e16462031d970b8a927d0e265e441a99f70203e0a2e0428add445970e257f4c536689e60c

$ echo "message" | subkey sign "favorite liar zebra assume hurt cage any damp inherit rescue delay panic"
50d830ac6ce90207584fcd46ada1db06d20db46370e440ae326e71c602d03641747031d66a9514b95a28462166336244bd401f25eb5e2e27b3f9dd5457b80d02

Проверка

Хотя вышеуказанные подписи отличаются друг от друга, они оба проверяются с адресом:

$ echo "message" | subkey verify 501...60c 5EXWNJuoProc7apm1JS8m9RTqV3vVwR9dCg6sQVpKnoHtJ68
Signature verifies correctly.

$ echo "message" | subkey verify 50d...d02 5EXWNJuoProc7apm1JS8m9RTqV3vVwR9dCg6sQVpKnoHtJ68
Signature verifies correctly.
